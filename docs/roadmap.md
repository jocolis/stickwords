# StickWords Roadmap

## 项目目标

StickWords 是一个运行在 M5Stick C Plus + PC 后端上的轻量背单词系统：

- M5Stick C Plus 作为前端复习设备。
- PC 作为后端，负责词库、复习算法、网页管理和同步。
- 词库以 CSV 文件持久化保存。
- 支持 Wi-Fi 日常同步，USB 串口用于首次配置和故障恢复。
- 支持中断后继续、短期离线复习、异常断电恢复和上一题重新评分。

公开项目命名避免使用 `Anki`，只在描述中说明使用 spaced repetition 思路。

## 第一版边界

第一版要做：

- PC 负责轻量 SM-2 复习算法。
- M5Stick 只负责显示卡片、收集评分、缓存任务和上传结果。
- 网页管理页负责 PC 端所有日常业务操作。
- M5Stick 默认横屏使用，支持左右手横握自动旋转。
- 复习卡片包含英文单词、中文释义和例句。
- 支持到期复习词 + 每日固定数量新词。

第一版不做：

- 多用户账号系统。
- 多词书系统。
- M5Stick 端编辑词库。
- M5Stick 端完整计算复习计划。
- 长期离线后的复杂双向合并。
- 完整复刻 Anki 的所有学习步长。

## 阶段 0：项目初始化与设计文档

目标：

- 写清楚 roadmap。
- 写清楚系统设计。
- 明确硬件按键、通信方式、数据字段和可靠性边界。

验收：

- `docs/roadmap.md` 存在。
- `docs/design.md` 存在。
- 关键设计选择已经记录，后续开发不用靠口头记忆恢复。

## 阶段 1：PC 后端核心

目标：

- 实现 `data/vocab.csv` 读写。
- 实现 CSV 批量导入。
- 实现重复单词更新规则。
- 实现轻量 SM-2 复习算法。
- 实现今日任务生成。
- 实现 `review_event_id` 幂等处理，避免重复上传导致重复计数。

验收：

- 可以用测试数据生成今日任务。
- `forgot / hard / good` 三种评分会正确更新复习状态。
- 重复导入同一个 `word` 时，只更新 `meaning / example`，不覆盖复习进度。
- 重复提交同一个 `review_event_id` 不会重复增加 `review_count`。

## 阶段 2：PC 网页管理页

目标：

- 提供 `start_stickwords.bat`，双击后启动后端并打开网页管理页。
- 网页管理页作为 PC 端控制台。
- 支持添加、编辑、停用单词。
- 支持 CSV 批量导入。
- 支持查看词库数量、今日待复习数量、最近同步状态。
- 显示 M5Stick 应使用的局域网地址。
- 预留 USB 配置向导入口。

验收：

- 用户只需要打开网页管理页，就能完成 PC 端主要业务操作。
- 后端关闭后重新运行，已有 CSV 数据仍能加载。
- 网页能显示建议写入 M5Stick 的 `server_url`。

## 阶段 3：M5Stick UI 原型

目标：

- 横屏显示界面。
- 区分 Button A、Button B 和电源键。
- 实现主菜单。
- 实现单线复习流程：
  - 单词页
  - 释义 + 例句摘要页
  - 完整例句页
  - 评分页
- 支持左右倾斜切换评分。
- 支持双摇直接提交“记住”。
- 支持左右手横握自动旋转。

验收：

- Button A 短按始终表示继续或确认。
- Button B 短按始终表示返回或重新评分。
- 电源键不绑定应用操作。
- 复习流程不需要复杂分叉。

## 阶段 4：Wi-Fi HTTP 同步

目标：

- M5Stick 通过 Wi-Fi 访问 PC 后端。
- 实现健康检查。
- 实现今日任务下载。
- 实现复习结果上传。
- 实现当天任务缓存。
- 实现待上传评分队列。

验收：

- M5Stick 可以从 PC 获取今日任务。
- PC 关闭期间，M5Stick 可以继续复习已缓存任务。
- PC 重新启动后，M5Stick 可以上传待上传结果。

## 阶段 5：可靠性机制

目标：

- 支持正常退出后继续。
- 支持异常断电后继续。
- 支持上一题重新评分。
- 支持本地待上传队列持久化。
- 使用双槽缓存和 checksum 降低断电写入损坏风险。

验收：

- 复习中途退出后，再次进入可以选择继续。
- 电量耗尽或断电后，重新开机能恢复到最近保存的会话状态。
- 评分后进入下一词时，Button B 可以回到上一词评分页重新评分。
- 重新评分会覆盖本地待上传队列中的上一条结果。

## 阶段 6：USB 配置与救援

目标：

- USB 串口用于首次配置和故障恢复。
- 支持设置 Wi-Fi。
- 支持设置 PC 后端地址。
- 支持读取设备状态。
- 支持重置设备配置。
- 网页管理页提供 USB 配置向导。

验收：

- PC 局域网 IP 变化后，可以通过网页 + USB 重新写入 `server_url`。
- Wi-Fi 不可用时，USB 可以帮助诊断当前设备配置。
- USB 不承担日常完整词库同步。

## 阶段 7：体验优化

目标：

- 增加统计页面。
- 增加复习历史查看。
- 增加导出和备份。
- 增加路由器固定 IP 使用指南。
- 评估 UDP 自动发现后端。

验收：

- 用户可以看到学习进度和最近复习情况。
- 数据可以方便备份。
- 后端地址配置比第一版更省心。

## 文档机制

开发期间持续维护：

- `docs/roadmap.md`：阶段路线和范围边界。
- `docs/design.md`：系统设计和已确认决策。
- `docs/handoff.md`：阶段交接、启动方式、测试方式和已知限制。
- `CHANGELOG.md`：公开版本变更记录。
- 内部开发日志保存在本地忽略目录，不作为公开仓库文档发布。

每完成一个阶段，至少在公开文档或内部日志中记录：

- 日期。
- 完成内容。
- 测试结果。
- 遇到的问题。
- 解决方式。
- 下一步。
