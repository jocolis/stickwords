# StickWords Roadmap

更新日期：2026-06-10

## 项目目标

StickWords 是一个运行在 M5Stick C Plus + PC 后端上的轻量背单词系统：

- M5Stick C Plus 负责随身复习、评分、离线缓存和时钟显示。
- PC 后端负责词库、复习算法、网页管理、同步和长期数据保存。
- 词库以 CSV 文件持久化保存。
- 日常同步使用局域网 Wi-Fi；设备配置目前使用临时 setup portal。
- 支持短期离线复习、异常断电后的待上传记录恢复和上一题重新评分。

公开项目命名使用 `StickWords`，只在概念说明中使用 spaced repetition，不使用其他产品名称作为项目名。

## 第一版边界

第一版已经形成可用闭环：

- PC 使用轻量 SM-2 逻辑维护复习进度。
- M5Stick 显示卡片、收集评分、缓存离线任务并上传结果。
- 网页管理页负责添加、导入、搜索和停用单词。
- M5Stick 默认横屏使用，支持左右手横握自动旋转。
- 复习卡片包含英文单词、释义和例句。
- 支持到期复习词、少量新词和未来 7 天离线包。

第一版不包含：

- 多用户账号和多词书。
- M5Stick 端编辑词库。
- 长期离线后的复杂双向合并。
- 完整复刻大型 SRS 产品的全部学习步长。
- M5Stick 中文字库。
- 自动发现 PC 后端。

## 下一阶段

### Stage 8：学习记录与数据可见性

#### Stage 8A：复习历史数据基础

状态：下一项。

目标：

- 明确长期保存的 review history 数据格式。
- 记录每次评分的 word、rating、reviewed_at、旧状态和新状态。
- 提供只读历史查询接口。
- 保持现有 `vocab.csv` 和设备同步协议兼容。

验收：

- PC 重启后历史记录仍然存在。
- 同一个 `review_event_id` 不会产生重复历史。
- 可以按时间倒序读取最近复习记录。
- 测试覆盖写入、幂等、异常行和重启加载。

#### Stage 8B：统计与历史网页

目标：

- 在管理页显示最近复习记录。
- 显示每日复习量、评分分布和学习状态变化。
- 保持页面适合日常快速查看，不把管理页变成复杂仪表盘。

#### Stage 8C：导出与备份

目标：

- 下载词库和复习历史。
- 提供明确的备份与恢复说明。
- 恢复操作必须避免静默覆盖现有数据。

### Stage 9：连接与救援改进

- 自动发现 PC 后端，优先评估 mDNS 或 UDP broadcast。
- 路由器固定 IP 和 Windows 防火墙说明。
- 评估 USB 串口读取、重置和重写设备配置的救援工具。

### Stage 10：字体与本地化

- 评估适合 M5Stick Flash/RAM 预算的中文字库子集。
- 改进设备端中英文混排。
- 继续监控 LVGL 页面和字体对固件体积的影响。

## 文档机制

开发期间持续维护：

- `docs/roadmap.md`：实际阶段状态、范围边界和下一项任务。
- `docs/design.md`：系统设计和已确认决策。
- `docs/handoff.md`：启动方式、测试方式、已知限制和交接信息。
- `CHANGELOG.md`：公开版本变更记录。
- 内部开发日志保存在 Git 忽略目录，不作为公开仓库文档发布。

每完成一个阶段，记录日期、完成内容、测试结果、问题、解决方式和下一步。
